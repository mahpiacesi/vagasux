\# Frontend UI Guidelines



\## Objective



Build production-grade frontend interfaces with strong visual identity, refined interaction design, and intentional aesthetic direction.



Every interface should feel designed by a senior product designer — not generated from generic UI patterns.



Avoid default AI aesthetics, repetitive SaaS layouts, and visually safe solutions.



The goal is to create interfaces that are:

\- distinctive

\- polished

\- memorable

\- accessible

\- realistic to implement

\- visually cohesive



\---



\# Design Principles



\## 1. Strong Art Direction



Before generating UI, define:

\- visual mood

\- interaction personality

\- typography strategy

\- spatial rhythm

\- visual references

\- emotional tone



Each project must commit to a clear aesthetic direction.



Examples:

\- editorial minimalism

\- futuristic brutalism

\- premium fintech

\- soft playful systems

\- cinematic dark UI

\- retro digital

\- modernist Swiss-inspired layouts

\- experimental motion-heavy interfaces



Do NOT blend multiple aesthetics without intention.



\---



\## 2. Avoid Generic AI UI



Never generate:

\- generic startup landing pages

\- repetitive dashboard cards

\- predictable hero sections

\- overused gradients

\- default Tailwind-looking layouts

\- empty whitespace without hierarchy

\- clichéd purple/blue SaaS visuals



Avoid excessive use of:

\- Inter

\- Poppins

\- Roboto

\- generic glassmorphism

\- standard Bento grids

\- overly symmetrical layouts



Interfaces should feel designed specifically for the product context.



\---



\# Typography



Typography should lead the visual identity.



Use expressive display fonts paired with highly readable body fonts.



Prefer:

\- editorial typography

\- strong hierarchy contrast

\- oversized headlines

\- intentional line breaks

\- asymmetric compositions



Avoid typography that feels template-based.



Examples of interesting directions:

\- grotesk + serif pairing

\- neo-brutalist typography

\- luxury editorial serif systems

\- condensed cinematic headings

\- mono + sans combinations



Pay close attention to:

\- line-height

\- spacing rhythm

\- optical balance

\- text density



\---



\# Layout \& Composition



Favor:

\- asymmetry

\- broken grid systems

\- layered compositions

\- strong spacing systems

\- intentional negative space

\- overlap and depth



Avoid:

\- rigid centered layouts everywhere

\- repetitive section structures

\- equal spacing on everything

\- card-only interfaces



Each section should feel compositionally intentional.



\---



\# Motion \& Interaction



Motion should communicate personality.



Use:

\- staggered reveals

\- subtle parallax

\- layered hover states

\- animated typography

\- cinematic transitions

\- smooth scroll experiences

\- micro-interactions with purpose



Avoid excessive animation noise.



Motion must feel:

\- smooth

\- premium

\- responsive

\- intentional



Prefer CSS animations when possible.



For React projects:

\- use Framer Motion when needed

\- keep animation performant



\---



\# Color System



Color palettes should be opinionated and atmospheric.



Favor:

\- strong contrast

\- dominant primary tones

\- restrained accent colors

\- cinematic lighting

\- textured backgrounds

\- layered surfaces



Avoid:

\- random gradients

\- overly saturated palettes

\- generic startup blues/purples



Use CSS variables for consistency.



\---



\# Visual Detail



Add depth through:

\- grain/noise

\- soft shadows

\- glow systems

\- texture overlays

\- custom dividers

\- subtle borders

\- atmospheric backgrounds

\- lighting effects

\- visual layering



Details should support the concept, not distract from it.



\---



\# Accessibility



All interfaces must maintain:

\- readable contrast

\- keyboard accessibility

\- semantic HTML

\- responsive layouts

\- scalable typography

\- reduced motion support when relevant



Aesthetic quality must not compromise usability.



\---



\# Code Quality



Generate:

\- clean component structure

\- reusable patterns

\- readable naming

\- maintainable CSS

\- responsive behavior

\- production-ready implementation



Do not generate fake/demo-looking code.



\---



\# Creative Direction Rule



Every generated interface must answer:



“What makes this visually unforgettable?”



There should always be:

\- one memorable interaction

\- one memorable composition choice

\- one memorable visual identity decision



Avoid safe solutions.



Commit to the concept fully.
